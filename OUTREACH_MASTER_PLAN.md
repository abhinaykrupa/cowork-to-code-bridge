**"Bridge Claude Code subscription as a local MCP provider for your agents — no API keys, no separate billing, full local context."**

- **LangChain/LangGraph:** "Cost-effective local execution backend without separate API billing"

✅ Idempotent + crash-safe

## FAQ

**Q: Why post to so many repos?**  
A: Every framework that supports agent orchestration is a potential user. The bridge solves a real problem for each (local execution without API billing). Breadth = visibility.

**Q: Should we expect responses from all repos?**  
A: No. Official frameworks (Microsoft, OpenAI) will likely respond. Community frameworks may or may not. Success metric is visibility + mutual linking, not 100% response rate.

**Q: How long should we wait between posting phases?**  
A: Phase 2A (official frameworks) should go out first — they get priority. Phase 2B/2C can follow after 3-5 days. Phases 3-4 can overlap with Phase 2.

**Q: Should we mention the bridge in existing related issues?**  
A: No. Opening new issues is cleaner + clearer. Only mention in threads if directly relevant and bridge isn't already visible in repo.

**Q: What if a repo already has a similar solution?**  
A: Acknowledge it in the issue, explain why bridge is complementary (local-first, zero network, file-based). Still post.
